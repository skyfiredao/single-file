# 天聊 / SkyChat

天聊是一个自托管、加密且无外部依赖的 Web 聊天应用。

**[English](README.md)**

许可证：GPL v3

## 功能特性

- **加密聊天**：所有消息均使用 AES-256-GCM 进行加密，服务器仅存储加密后的密文。
- **URL 散列分发密钥**：房间密钥通过 URL 散列片段（Hash Fragment）分发，永远不会发送到服务器。
- **三层密钥体系**：
    - **房间密钥 (Room Key)**：AES-256-GCM 对称密钥，用于加密消息。
    - **身份密钥 (Identity Key)**：每个成员拥有一个 ECDSA P-256 密钥对，用于对消息进行签名。
    - **管理员密钥 (Admin Key)**：房间创建者的 ECDSA P-256 密钥对，用于执行管理操作。
- **极简无依赖**：
    - **后端**：纯 Python 3.10+ 环境，仅使用标准库（`asyncio` 和 `sqlite3`）。无需 Flask、Django 或任何第三方库。
    - **前端**：单个 `index.html` 文件，包含所有 HTML、CSS 和 JavaScript。无需 React、Vue、npm 或 Node.js。
- **双语支持**：内置中英文双语界面。
- **电报风格 UI**：现代聊天气泡设计，配有根据昵称哈希值生成的确定性颜色圆头像。
- **临时账户**：只需输入昵称和密码即可使用，无需繁杂的注册流程。
- **房间有效期**：房间会在设定的时长（1小时、6小时、24小时、72小时）后自动失效。
- **管理员权限**：房间创建者可以清空聊天记录或直接关闭房间。
- **轮询架构**：采用 3 秒间隔的 HTTP 轮询配合时间戳锚点实现增量更新。不使用 WebSocket。
- **安全加固**：包含单 IP/操作频率限制、SHA-256 两次加盐哈希存储密码、每次请求生成唯一 CSP nonce、AES-GCM 附加认证数据绑定、以及防篡改消息签名。
- **会话恢复**：身份信息存储在 `localStorage` 中，并受 PBKDF2 生成的密钥保护。
- **移动端适配**：完美适配桌面及移动浏览器。
- **便捷邀请**：通过可分享的 URL 直接邀请好友加入。

## 快速上手

1. 克隆仓库：
   ```bash
   git clone <repo-url>
   cd skychat
   ```
2. 运行服务器：
   ```bash
   python3 server.py
   ```
3. 在浏览器中打开 `http://<服务器IP>:8080`

## 使用说明

- **创建房间**：点击“创建新房间 / Create Room”，设置昵称和密码，并选择房间有效期。
- **邀请他人**：点击“链接 / Link”按钮复制邀请链接并发送给好友。
- **加入房间**：通过浏览器打开邀请链接，输入昵称和密码。
- **发送消息**：在页面底部输入文字，点击“发送 / Send”或按回车。
- **手动刷新**：点击 🔄 按钮可手动同步最新的消息和成员列表。
- **退出房间**：点击“退出 / Exit”按钮。
- **管理功能**：房间创建者会看到“清空 / Clear”和“关闭 / Close”按钮。

## 命令行选项

```bash
python3 server.py [--host HOST] [--port PORT] [--db DB_PATH]
```

| 参数 | 默认值 | 说明 |
|------|---------|-------------|
| `--host` | `0.0.0.0` | 绑定地址 |
| `--port` | `8080` | 监听端口 |
| `--db` | `chat.db` | SQLite 数据库路径 |

仅本机调试：
```bash
python3 server.py --host 127.0.0.1
```

## 系统架构

- **精简代码**：整个项目由两个核心文件组成：`server.py`（约 855 行）和 `index.html`（约 1370 行）。
- **数据持久化**：使用 SQLite（WAL 模式）存储消息，首次运行时会自动创建 `chat.db`。
- **无外部依赖**：严格依赖 Python 3.10+ 标准库。
- **端侧加密**：所有加解密操作均在浏览器内通过 Web Crypto API 完成。服务器无法接触明文消息、昵称或房间密钥。
- **同步机制**：客户端每 3 秒通过时间戳锚点向服务器请求增量更新。

## 安全模型

天聊通过以下三层密钥保障安全：
- **房间密钥 (AES-256-GCM)**：共享对称密钥，用于加密房间内所有消息。通过 URL 散列传递，不经过服务器。
- **身份密钥 (ECDSA P-256)**：用户生成的密钥对，用于消息签名。私钥由用户密码通过 PBKDF2 算法保护。
- **管理员密钥 (ECDSA P-256)**：创建者的专属密钥对，用于验证管理指令。

**安全加固措施**：
- **CSP Nonce**：每次页面加载生成唯一 `nonce`，用于 Content-Security-Policy 的 `script-src` 指令，阻止注入脚本执行。
- **AES-GCM AAD**：消息加密时使用发送者的成员 ID 作为附加认证数据（AAD），将密文与发送者绑定，防止消息被重新归属。
- **签名绑定**：数字签名覆盖 `密文 + IV + 发送者ID`（而非明文），确保传输过程中的篡改可被检测，同时不暴露消息内容。
- **无 innerHTML**：所有 DOM 更新使用安全 API（`textContent`、`createElement`、`removeChild`），消除 XSS 攻击向量。
- **频率限制**：基于 IP 和操作类型的频率限制，防止暴力破解和滥用。
- **密码哈希**：密码在客户端先经过 SHA-256 哈希，再在服务器端使用随机盐进行二次哈希。服务器永远不会接触明文密码。
- **安全响应头**：X-Frame-Options (DENY)、X-Content-Type-Options (nosniff)、Referrer-Policy (no-referrer)、Permissions-Policy。

**可见性说明**：
- **服务器可见**：房间 ID、成员人数、消息总数、时间戳和成员公钥。
- **服务器不可见**：消息内容、用户昵称、房间密钥、用户密码以及私钥。

**局限性与注意事项**：
- **URL 即密钥**：任何拥有邀请链接的人都能获取房间密钥。请仅通过可信渠道分享链接。
- **无前向安全性**：若房间密钥泄露，所有历史消息均可被解密。不支持单条消息密钥轮换。
- **共享房间密钥**：所有成员使用同一对称密钥。一个成员被攻破意味着该房间所有消息可被解密。
- **localStorage 身份存储**：用户身份保存在浏览器 `localStorage` 中。清除浏览器数据将永久丢失该房间的身份信息，且无法恢复。
- **同浏览器限制**：由于共享 `localStorage`，无法在同一浏览器中模拟两个用户。测试时请使用不同浏览器或隐私/无痕窗口。
- **不支持文件/图片**：仅支持纯文本消息，不支持文件上传、图片或富媒体。
- **房间容量**：每个房间最多 50 人。
- **消息长度**：每条消息最长 10,000 个字符。
- **轮询而非实时**：消息通过每 3 秒一次的 HTTP 轮询获取，非即时推送，最长可能有 3 秒延迟。
- **无离线消息**：服务器重启后数据库仍存在，但客户端需重新加入（刷新页面即可）。
- **服务器信任**：虽然服务器无法读取消息内容，但恶意的服务器运营者可篡改公钥、屏蔽消息或记录元数据（IP、时间戳、房间活动规律）。
- **无内置 TLS**：服务器不提供 HTTPS。生产环境建议通过反向代理（如 Nginx、Caddy）启用 TLS。
- **SQLite 并发**：SQLite WAL 模式可处理适度并发，但不适合同时在线数百用户的高吞吐量场景。
- **依赖浏览器加密 API**：所有加密操作依赖 Web Crypto API，不支持该 API 的旧版浏览器无法使用。

## 技术栈

| 组件 | 技术方案 |
|-----------|-----------|
| 后端 | Python 3 标准库 (asyncio + sqlite3) |
| 前端 | HTML + CSS + JavaScript (单文件) |
| 加密 | Web Crypto API (AES-256-GCM, ECDSA P-256, PBKDF2) |
| 存储 | SQLite (WAL 模式) |
| 依赖 | 无 |
