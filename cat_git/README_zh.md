# 🔀 Cat Git

**零依赖、单文件的浏览器端 Git 仓库浏览工具，完全由 [OpenCode](https://github.com/anomalyco/opencode) 编写。**

[English](README.md)

## 命名小故事

Cat Git 源于一个简单的问题：*能不能不装任何软件，直接在浏览器里查看 Git 历史？*

它是 [Cat Doc](../cat_doc/)（单文件文档管理器）的伴侣工具。Cat Doc 负责文档的阅读和编辑，但无法查看 `doc/` 工作目录下 Git 仓库中的版本历史。

于是有了这个想法：把一个完整的 Git 引擎嵌入到单个 HTML 文件里，让它遍历提交图谱、显示 diff——全部离线运行，全部在客户端完成。`isomorphic-git` 库让这成为可能：一个完整的 JavaScript Git 实现，可以直接在浏览器中运行。

整个项目的每一行代码都是通过 [OpenCode](https://github.com/anomalyco/opencode) 完成的——一个 AI 编程助手。人类提供方向，OpenCode 负责工程实现。

## 功能概览

Cat Git 是一个**单 HTML 文件**（约 84KB 应用逻辑 + 内嵌 isomorphic-git），让你在浏览器中浏览本地 Git 仓库、查看文件历史、比较差异、检出历史版本——全部在浏览器中完成。

![Cat Git 截屏](../screenshot/cat_git.jpg)

### 核心功能

| 功能 | 说明 |
|------|------|
| **单文件架构** | 一个 `cat_git.html`——放到任意目录，浏览器打开，即用 |
| **自动检测 Git 仓库** | 扫描 `doc/` 目录，列出所有发现的 Git 仓库 |
| **仓库与分支切换** | 下拉菜单快速选择仓库和分支 |
| **延迟加载文件树** | 逐级展开目录——高效处理大型仓库 |
| **文件级提交历史** | 仅显示修改了选中文件的提交，而非仓库全局日志 |
| **优化的历史算法** | 从 HEAD 回溯父提交链，比较 blob OID 并使用 `treeWalkCache`——避免扫描完整提交历史 |
| **渐进式渲染** | 版本历史条目在查找过程中逐步显示 |
| **左右分栏 diff** | Myers diff 算法，包含上下文行（±3 行）和未变更区域折叠显示 |
| **版本检出** | 将文件强制还原到任意历史版本（`git checkout --force`，需二次确认） |
| **深度链接到 Cat Doc** | 通过 URL hash 在 Cat Doc 中打开文件，享受完整语法高亮 |
| **自动恢复** | 通过 IndexedDB 记住已授权的目录——无需重复授权 |
| **双语界面** | 所有标签中文附英文翻译 |

### 版本历史条目显示

右侧面板中每个版本条目包含：

| 字段 | 样式 |
|------|------|
| 提交时间 | 黑色粗体 |
| 提交哈希（短） | 黑色普通字重 |
| 作者名（仅取第一个词） | 黑色粗体，在第二行 |
| 提交信息 | 普通字重，与作者同行 |

条目使用交替背景色增强视觉层次，悬停和选中时显示白色高亮。

## 快速开始

### 1. 部署

把 `cat_git.html` 放在 `doc/` 目录旁边即可，不需要服务器。

```bash
my-workspace/
├── cat_git.html
├── cat_doc.html    # 可选——用于语法高亮的文件查看
├── cat_md.html     # 可选——轻量 Markdown 查看器
└── doc/
    ├── my-repo/    # 一个 Git 仓库
    └── notes/      # 普通文件
```

### 2. 授权

首次打开时，点击 **"授权访问 (Authorize)"**。浏览器会要求选择父目录。Cat Git 自动使用其中的 `doc/` 子目录并扫描 Git 仓库。

### 3. 浏览

- **顶部栏**：从下拉菜单选择仓库和分支。
- **左侧栏**：浏览文件树。点击展开目录，点击文件查看。
- **中间面板**：文件内容显示。
- **右侧面板**：选中文件的提交历史。点击某个版本查看与当前版本的 diff。
- **版本检出**：查看 diff 后，点击"切换到此版本 (Checkout)"还原文件（需确认）。

## 架构设计

```
cat_git.html（单文件）
├── <style>        — CSS 变量、布局、淡绿色主题、diff 样式
├── <script>       — isomorphic-git（内嵌，完整的 JavaScript Git 引擎）
├── <script>       — lightning-fs（内嵌，用于 Git 操作的内存文件系统）
└── <script>       — 应用逻辑
    ├── 安全模块    — 目录验证、路径约束
    ├── 状态管理    — 目录句柄、仓库句柄、分支、选中文件
    ├── IndexedDB  — 句柄持久化，实现自动恢复
    ├── 初始化      — 自动恢复或显示欢迎页
    ├── 仓库扫描    — 检测 doc/ 下的 .git 目录
    ├── 文件树      — 延迟加载目录展开、文件树渲染
    ├── 版本历史    — 父提交链回溯算法 + treeWalkCache
    │              — 渐进式渲染提交条目
    ├── Diff       — Myers diff 算法，左右分栏 HTML 渲染
    ├── 版本检出    — 强制检出到历史版本并重置
    └── Cat Doc    — 通过 URL hash 深度链接集成
```

### 设计决策

- **单文件**：无构建步骤、无打包器、无 CDN——离线可用，随处可用
- **内嵌 isomorphic-git**：纯 JavaScript 的完整 Git 实现，无需服务器
- **自定义历史算法**：标准 `git.log({ filepath })` 在大型仓库中太慢——使用父提交链回溯 + blob OID 对比 + 树遍历缓存
- **`doc/` 子目录锁定**：所有操作限制在 `doc/` 内——防止意外访问系统文件
- **无远程操作**：克隆和拉取必须在外部完成——保持工具简洁和安全
- **淡绿色主题**：与 Cat Doc 的暖黄配色视觉区分

## 与 Cat Doc、Cat md 的关系

Cat Git 是 Cat 工具集的一部分，但每个工具完全独立：

| 方面 | 说明 |
|------|------|
| **相互独立** | Cat Git、Cat Doc、Cat md 都是独立的单文件工具——互不依赖 |
| **链接导航** | 通过顶部导航栏相互链接 |
| **共享工作空间** | 所有工具都以 `doc/` 目录为工作根目录 |
| **深度链接** | Cat Git 通过 URL hash 在 Cat Doc/Cat md 中打开文件（例如 `cat_doc.html#file=path`） |

## 技术要点

- **嵌入式 Git 引擎**：`isomorphic-git` 提供 `git.log`、`git.readBlob`、`git.resolveRef`、`git.walk` 等功能——全部在客户端运行
- **历史性能优化**：避免 O(N) 全量日志扫描；直接回溯父提交链并比较 tree/blob OID，使用 `treeWalkCache` 去重
- **Diff 引擎**：内置 Myers diff 算法生成左右对比视图，包含上下文行和折叠标记——无需外部库
- **无网络依赖**：一切在本地运行；仓库必须预先通过外部工具克隆

## 已知局限

- **仅 Chromium**：依赖 File System Access API——不支持 Safari/Firefox
- **无远程操作**：不能克隆、拉取或推送——仓库必须本地存在
- **无合并/变基**：仅支持浏览、diff 和单文件检出
- **文件大小**：不支持大型二进制文件
- **性能**：超大型仓库（10 万+ 提交）历史加载可能较慢
