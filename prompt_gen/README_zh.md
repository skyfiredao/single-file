## 命名小故事

"提示词优化" — 把一个粗略的想法交给本地大模型，配合预设的优化策略，输出一段打磨好的、细节丰富的提示词，拿来就能用。

这个项目源于一个简单的需求：别再手动改写提示词了，让本地 AI 来干 —— 全程离线，无需云端，无需 API 密钥，数据不出本机。

整个项目的每一行代码都是通过 [OpenCode](https://github.com/anomalyco/opencode) 完成的。人类提供构思，OpenCode 负责工程实现。

## 功能概览

提示词优化是一个**单 HTML 文件**（约 84KB，约 2080 行），可连接本地 LLM 服务器（Ollama、LM Studio 或任何 OpenAI 兼容端点）来优化和扩展用户提示词。它采用中英双语界面，左右面板布局：左侧输入原始提示词，右侧实时流式输出优化结果。

![提示词优化截屏](../screenshot/prompt_gen.jpg)

### 核心功能

| 功能 | 说明 |
|------|------|
| **单文件架构** | 一个 `prompt_gen.html`——浏览器打开，即用 |
| **多供应商支持** | 管理多个 LLM 供应商，可自定义名称、端点、API 令牌和 API 类型；支持局域网地址连接远程服务器 |
| **模型选择** | 自动从活跃供应商获取可用模型列表；记住上次选择 |
| **预设系统** | 可自定义的系统提示词，定义优化策略（如扩展示例、改写提炼等） |
| **预设排序** | 通过 ⏫🔼🔽⏬ 按钮调整顺序——置顶、上移、下移、置底 |
| **输出语言** | 可选 Auto、中文 或 English 控制优化结果的输出语言 |
| **可编辑输出** | 右侧面板的输出内容可直接编辑，在原位精修结果 |
| **关闭思考模式** | 设置中的开关，启用后自动在用户消息前插入 `/no_think` 指令，跳过 Qwen3 等 thinking model 的推理阶段 |
| **字数统计** | 输入框（最多 2500 字符）和输出框（最多 8000 字符）均有实时字数统计 |
| **复制/粘贴/清空** | 一键剪贴板操作，带气泡提醒反馈；优化按钮旁有独立的绿色**复制**按钮，方便快速复制结果 |
| **流式输出** | 通过 SSE 实时逐 token 流式输出；支持 OpenAI 和 Ollama 两种流格式，30 秒空闲超时（仅在持续无数据时触发——活跃生成永不超时） |
| **优化历史** | 通过 localStorage 自动保存 100 条历史，支持恢复和删除 |
| **设置持久化** | 所有配置均保存在本地；记住上次使用的供应商、模型、预设和输出语言 |
| **配置导入导出** | 导出全部设置和历史为 JSON 文件；导入可恢复或在不同机器间转移配置 |
| **重置配置** | 设置中一键重置，清除所有数据恢复默认值 |
| **自适应文本框** | 输入框随内容自动伸展，无滚动条 |
| **示例提示词** | 输入框预填一句示例文字，打开即可体验 |
| **加载动画** | 流式输出或等待优化响应时显示闪动省略号 |
| **中英双语界面** | 所有标签和提示均为中英双语显示 |
| **CORS 引导** | 以 `file://` 协议打开时，内置横幅提供修复说明 |
| **响应式设计** | 桌面端左右布局（1:2 比例），手机端（<768px）垂直布局 |
| **错误处理** | 清晰的 API 失败、CORS 问题、空响应或模型缺失提醒 |

### 安全加固

| 措施 | 说明 |
|------|------|
| **内容安全策略 (CSP)** | CSP 元标签允许 http/https 连接至任意端点（支持 localhost、局域网和远程服务器）；URL 安全由 `sanitizeUrl()` 在应用层面强制执行；阻止所有外部资源加载 |
| **XSS 防护** | 所有用户输入通过 `textContent` 渲染；SVG 图标通过 DOM API (`createElementNS`) 创建；错误显示使用 `escapeHtml()` 转义 |
| **URL 校验** | 所有端点通过 `sanitizeUrl()` 验证，仅接受 http/https 协议 |
| **原型污染防护** | `pickKnownKeys()` 白名单过滤所有外部数据（localStorage 加载、JSON 导入），仅接受已知配置键 |
| **数据验证** | 导入的供应商、预设和历史条目均经过长度限制验证（token ≤500、userPrompt ≤2.5K、optimizedPrompt ≤8K） |
| **请求超时** | CORS 检测（5s）和模型列表（15s）使用 `fetchWithTimeout()`；优化请求使用 `fetchStreaming()` 配 30s 空闲超时——仅在连续 30 秒无数据时触发；活跃流式输出永不超时 |
| **响应防御** | API 响应使用安全属性链检查；空结果主动抛出明确错误 |
| **导入大小限制** | JSON 导入上限 5MB；处理前验证文件格式和 `app` 字段 |
| **Referrer 策略** | `no-referrer`——防止 URL 泄露给外部服务器 |
| **默认值冻结** | 默认预设和供应商使用 `Object.freeze()`，运行时不可篡改 |
| **Unicode 安全** | `safeStringSlice()` 使用 `Array.from` 防止历史显示中截断代理对（surrogate pair） |
| **密码管理器抑制** | Token 输入框使用 CSS 文本遮罩代替 `type="password"`，避免浏览器弹出保存密码提示；配合 `autocomplete="off"` 及扩展忽略属性 |

## 前置要求

使用提示词优化需要在机器上运行本地大语言模型（LLM）服务器。

### 1. 本地 LLM 服务器
- **Ollama**（推荐）
- **LM Studio**
- 任何 **OpenAI 兼容** API 服务器

### 2. 配置 CORS（对 Ollama 至关重要）
浏览器打开本地 HTML 文件时会发送 `Origin: null` 头部，Ollama 默认会拦截此类请求。你必须手动允许所有来源以确保浏览器能够访问。

**macOS：**
```bash
# 设置环境变量允许所有来源
launchctl setenv OLLAMA_ORIGINS "*"
# 然后从菜单栏重启 Ollama 应用程序
```

**Linux：**
```bash
# 对于 systemd 管理的 Ollama，编辑服务文件
sudo systemctl edit ollama.service
# 在 [Service] 下添加：
# Environment="OLLAMA_ORIGINS=*"
sudo systemctl restart ollama
```

对于 **LM Studio**，CORS 通常默认开启。如果遇到错误，请检查侧边栏中的服务器设置。

## 快速开始

### 1. 启动服务器
确保你的 Ollama 或 LM Studio 已按"前置要求"中的说明启动并配置。

### 2. 打开应用
双击 `prompt_gen.html` 或在终端运行命令：

```bash
open prompt_gen/prompt_gen.html
```

### 3. 配置供应商
点击**齿轮图标**进入设置。默认已预配置 Ollama（`localhost:11434`）和 LM Studio（`127.0.0.1:1234`）两个供应商。可按需添加、编辑或删除供应商，也支持局域网地址（如 `192.168.x.x`）。每个供应商支持自定义名称、端点地址、API 令牌和 API 类型（OpenAI 兼容或 Ollama 原生）。

### 4. 选择模型
在主界面工具栏中，选择一个供应商，然后点击**刷新按钮**加载可用模型，选择你想使用的模型。

### 5. 开始优化
在左侧输入你的提示词（最多 2500 字符，已预填示例文字），选择预设策略和输出语言，然后点击**优化 Optimize**。优化结果在右侧实时流式输出——可编辑、可复制，带字数统计（最多 8000 字符）。点击优化按钮旁的绿色**复制 Copy**按钮可一键复制结果到剪贴板。

### 6. 管理预设（可选）
在设置中，找到**预设管理**。可添加新的优化策略（自定义系统提示词）、编辑现有预设、通过 ⏫🔼🔽⏬ 按钮调整顺序、或删除。

### 7. 导入导出（可选）
在设置中，使用**导出 Export** 将全部配置和历史保存为 JSON 文件。使用**导入 Import** 从备份恢复，或在不同机器间转移设置。

## 架构设计

```
prompt_gen.html（约 84KB，单文件）
├── <head>
│   ├── CSP 元标签             — 允许 http/https 连接；限制资源加载
│   └── Referrer 策略          — no-referrer
├── <style>
│   ├── CSS 变量               — 主题及一致的设计令牌
│   ├── 响应式布局             — Flexbox，1:2 面板比例，768px 移动端断点
│   ├── 加载动画               — @keyframes blink 省略号动画
│   └── 视觉样式               — 浅蓝色输入面板、浅绿色输出面板、阴影增强边框
└── <script>
    ├── 安全工具               — escapeHtml()、sanitizeUrl()、pickKnownKeys()、
    │                            fetchWithTimeout()、fetchStreaming()、safeStringSlice()、
    │                            createDeleteIcon()
    ├── 状态管理               — 配置、历史、上次预设/语言（均存在 localStorage）
    ├── 多供应商系统           — 增删改供应商，含名称、端点、令牌、API 类型
    ├── 接口集成               — OpenAI 兼容 (/v1/chat/completions) 与 Ollama 原生 (/api/chat)
    │                            支持两种格式的实时 SSE 流式输出
    ├── 模型管理               — 自动获取活跃供应商的模型列表，持久化选择
    ├── 预设系统               — 增删改查 + 排序（置顶/上移/下移/置底），系统提示词编辑器
    ├── 提示词优化             — 组合预设系统提示词 + 输出语言指令；
    │                            逐 token 流式输出响应，30s 空闲超时
    ├── 历史管理               — 保存、渲染、恢复、删除、清空（上限 100 条）
    ├── 导入导出               — 完整配置 + 历史导出为 JSON，导入时带验证
    ├── 数据验证               — validateProvider()、validatePreset()、validateHistoryItem()
    ├── UI 组件                — 设置/预设/供应商/历史弹窗、CORS 横幅、气泡提醒
    └── 剪贴板                 — 基于 navigator.clipboard API 的复制/粘贴
```

## 已知局限

- **仅限文本**：提示词输入限 2500 字符，输出限 8000 字符；不支持文件或图片类提示词。
- **服务器依赖**：需要运行本地 LLM 服务器（Ollama、LM Studio 或兼容端点）。
- **模型质量**：优化效果完全取决于所选的本地模型。
- **剪贴板权限**：`file://` 协议下每次粘贴可能弹出权限提示；使用 `http://localhost` 打开可永久授权。
