## 命名小故事

"爱翻译" — "爱" 的发音与 "AI" 完美契合。这是一个你会爱上的本地 AI 翻译工具。

该项目源于对隐私和离线翻译的需求。通过本地大语言模型（LLM）进行翻译，无需云端，无需 API 密钥，没有任何数据会离开你的机器。

整个项目的每一行代码都是通过 [OpenCode](https://github.com/anomalyco/opencode) 完成的。人类提供构思，OpenCode 负责工程实现。

## 功能概览

爱翻译是一个**单 HTML 文件**（约 52KB，约 1450 行），可连接本地 Ollama 或 LM Studio 服务器进行 AI 驱动的文本翻译。它采用了类似 Google 翻译的中英双语界面设计，支持多种语言对及自定义语言扩展。

![爱翻译截屏](../screenshot/ai_translate.jpg)

### 核心功能

| 功能 | 说明 |
|------|------|
| **单文件架构** | 一个 `index.html`——浏览器打开，即用 |
| **双后端支持** | 支持 Ollama 和 LM Studio，可配置接口地址 |
| **内置语言对** | 简体中文 ↔ English / 日本語 / 繁體中文 |
| **自定义语言** | 在设置中添加任意语言（如 Deutsch、Français、한국어） |
| **一键互换** | ⇄ 按钮互换源/目标语言及文本，并自动触发翻译 |
| **复制/粘贴/清空** | 一键剪贴板操作，带气泡提醒反馈；粘贴后自动触发翻译 |
| **字数限制** | 5000 字符限制，带实时统计 |
| **翻译历史** | 通过 localStorage 自动保存 100 条历史，支持恢复和删除 |
| **设置持久化** | 后端、地址、模型选择保存在本地，页面加载时自动恢复 |
| **默认语言对** | 每次刷新页面恢复为 English → 简体中文 |
| **切换自动翻译** | 切换语言选项卡时，若有源文本则自动触发翻译 |
| **重置配置** | 设置中一键重置，清除所有数据恢复默认值 |
| **自适应文本框** | 输入输出框随内容自动伸展，无滚动条 |
| **加载动画** | 等待翻译响应时显示闪动省略号 |
| **中英双语界面** | 所有标签和提示均为中英双语显示（语言选项卡除外） |
| **智能提示词** | 简繁转换使用专门优化的提示词（双向支持） |
| **响应式设计** | 桌面端左右布局，手机端（<768px）垂直布局 |
| **错误处理** | 清晰的 API 失败、CORS 问题或模型缺失提醒 |

### 安全加固

| 措施 | 说明 |
|------|------|
| **内容安全策略 (CSP)** | CSP 元标签限制仅允许连接 localhost 及局域网私有网段（10.x、172.16-31.x、192.168.x），阻止外部资源加载 |
| **XSS 防护** | 所有用户输入通过 `textContent`/`escapeHtml()` 转义，无原始 innerHTML 注入 |
| **URL 校验** | 后端地址通过 `sanitizeUrl()` 验证，仅接受 http/https 协议 |
| **无全局暴露** | 无不必要的函数挂载到 `window` 对象 |
| **Referrer 策略** | `no-referrer`——防止 URL 泄露给外部服务器 |
| **无内联事件** | 所有事件处理通过 `addEventListener` 绑定，无内联 `onclick` |

## 前置要求

使用爱翻译需要在机器上运行本地大语言模型（LLM）服务器。

### 1. 本地 LLM 服务器
- **Ollama**（推荐）
- **LM Studio**

### 2. 配置 CORS（对 Ollama 至关重要）
浏览器打开本地 HTML 文件时会发送 `Origin: null` 头部，Ollama 默认会拦截此类请求。你必须手动允许所有来源以确保浏览器能够访问。

**macOS：**
```bash
# 设置环境变量允许所有来源
launchctl setenv OLLAMA_ORIGINS "*"
# 然后从菜单栏重启 Ollama 应用程序
```

**Linux：**
```bash
# 对于 systemd 管理的 Ollama，编辑服务文件
sudo systemctl edit ollama.service
# 在 [Service] 下添加：
# Environment="OLLAMA_ORIGINS=*"
sudo systemctl restart ollama
```

对于 **LM Studio**，CORS 通常默认开启。如果遇到错误，请检查侧边栏中的服务器设置。

## 快速开始

### 1. 启动服务器
确保你的 Ollama 或 LM Studio 已按"前置要求"中的说明启动并配置。

### 2. 打开应用
双击 `index.html` 或在终端运行命令：

```bash
open ai_translate/index.html
```

### 3. 配置后端
点击**齿轮图标**进入设置。选择后端（Ollama 或 LM Studio），点击**刷新**加载可用模型，然后选择你想使用的模型。

### 4. 开始翻译
选择语言对，输入或粘贴文本，点击**翻译**。

### 5. 添加自定义语言（可选）
在设置中，找到**自定义语言**。输入语言名称（如 `Deutsch`、`Français`、`한국어`），点击 **+ Add**。新语言会立即出现在语言选择栏中。

## 架构设计

```
index.html（约 52KB，单文件）
├── <head>
│   ├── CSP 元标签          — 限制连接 localhost 及局域网私有网段
│   └── Referrer 策略       — no-referrer
├── <style>
│   ├── CSS 变量            — 主题及一致的设计令牌
│   ├── 响应式布局          — Flexbox，768px 移动端断点
│   ├── 加载动画            — @keyframes blink 省略号动画
│   └── 视觉样式            — 浅绿色输出面板、珊瑚色互换按钮
└── <script>
    ├── 安全工具            — escapeHtml()、sanitizeUrl()
    ├── 状态管理            — 配置、历史存在 localStorage；语言对每次加载恢复默认
    ├── 接口集成            — Ollama (/api/chat) 与 LM Studio (/v1/chat/completions)
    ├── 语言逻辑            — 内置 + 自定义语言、互换/切换/粘贴均自动翻译
    ├── 提示词工程          — 简繁转换专用提示词
    ├── 自定义语言          — 添加/删除，存储在 localStorage，运行时合并
    ├── 历史管理            — 保存、渲染、恢复、删除、清空（上限 100 条）
    ├── UI 组件             — 设置弹窗、历史面板、气泡提醒
    └── 剪贴板              — 基于 navigator.clipboard API 的复制/粘贴
```

## 已知局限

- **仅限文本**：不支持文档、图片或网页翻译。
- **手动选择**：不支持自动语言识别。
- **无语音支持**：不支持语音输入或输出。
- **服务器依赖**：需要运行本地 LLM 服务器（Ollama 或 LM Studio）。
- **模型质量**：翻译效果完全取决于所选的本地模型。
- **剪贴板权限**：`file://` 协议下每次粘贴可能弹出权限提示；使用 `http://localhost` 打开可永久授权。
